Conditional Logic

Core engine for show/hide, piping, and dynamic content

How It Works

The conditional logic system scans all elements with the .logic-config class, reads their data-* attributes, evaluates conditions, and applies the results. It runs automatically on page load and re-runs 50ms after any input change.

  1. Add logic-config element
    Place a hidden div with class logic-config inside your question container.
  2. Define data attributes
    Set data-cond, data-target, and other attributes to control behavior.
  3. Auto-execution
    The script auto-detects and runs on page load. No manual initialization needed.
XML Setup (Decipher) — Real Example from D1.txt
Example 1: Show/Hide + Piping 
<checkbox label="X2" atleast="1">
  <title>multi select: <span class="pipe1">test</span>
    <div class="logic-config"
         data-cond="X1_r1"
         data-pipe=".pipe1:X1"
         data-target="X2_r(1,2,3)">
    </div>
  </title>
  <row label="r1">one</row>
  <row label="r2">two</row>
  <row label="r3">three</row>
</checkbox>
Example 2: Numeric + SUM Condition 
<number label="X3" size="3">
  <title>number question:
    <div class="logic-config"
         data-cond="X1_r(1-3) and X2_r1"
         data-target="N_X3_r2">
    </div>
  </title>
  <row label="r1">one</row>
  <row label="r2">two</row>
</number>

<!-- SUM condition example -->
<text label="X4">
  <title>text question:
    <div class="logic-config"
         data-cond="SUM(X3_r(1-2)) gt 10">
    </div>
  </title>
</text>
Data Attributes Reference
AttributeTypeDescriptionExample
data-cond string Condition expression to evaluate Q1_r1 || Q1_r2
data-target string Target question(s) to show/hide Q2, Q3_r(1-3)
data-pipe string Pipe answer to target element #pipeTarget:Q1

Slider Component

Modern interactive range slider with external quota support

How It Works

The Slider component intercepts Decipher's standard <number> inputs and transforms them into beautifully styled HTML5 range sliders. It supports customizable min/max values and mutual exclusivity. You can now link an external checkbox question (e.g., a hidden quota funnel) to act as the mutual exclusivity driver by mapping the column classes.

  1. Map the CSS Classes
    Add ss:colClassNames="slider col1" to your number question, and ensure your external checkbox question shares the same col1 class structure.
  2. Call the Initialization Script
    Call sliderComponent({...}) inside the question's style block, passing your specific configuration and the optional checkboxQuestion target.
Decipher XML Setup Example (With External Quota Checkbox)
Slider + External Checkbox Setup 
<number label="CMT_CQ5" optional="0" size="3" verify="range(1,7)">
  <title>Please evaluate the following therapies:</title>

  <!-- Note the column mapping: slider + col1, col2, etc. -->
  <col label="c1" ss:colClassNames="slider col1"><b>Overall Efficacy</b></col>
  <col label="c2" ss:colClassNames="slider col2"><b>Overall Safety</b></col>
  <col label="c3" ss:colClassNames="slider col3"><b>Efficacy vs Safety</b></col>
  
  <row label="r1">OpenText</row>
  <row label="r2">Microsoft</row>

  <style label="gridSlider" name="question.after"><![CDATA[
  <script>
    sliderComponent({
        question: "#question_CMT_CQ5",
        checkboxQuestion: "#question_hQ_SALES_COMPANY_FUNNEL",
        min: 1,
        max: 7,
        mode: "col"
    });
  </script>
  ]]></style>
</number>

  <title>Hidden question for quota purpose</title>
  <col label="c1" ss:colClassNames="col1"><b>Overall Efficacy</b></col>

  <row label="r1">OpenText</row>

Configuration Options

Pass these parameters in the configuration object when calling sliderComponent({...}) to customize behavior.

ParameterTypeDefaultDescription
question* string "" jQuery selector for the primary number question (e.g., "#question_CMT_CQ5").
checkboxQuestion string "" Optional target for a separate checkbox question to handle column-based exclusivity (e.g., "#question_hQ_1"). If blank, it assumes checkboxes are internal to the primary question.
min number 1 The minimum allowed value on the slider.
max number 7 The maximum allowed value on the slider.
mode string "col" Mutual exclusivity mode. Use "col" to link with checkboxes in the same column, or "row" for the same row.

  autoSum({
    question : "#question_Sum1",
    mode     : "row",
    from     : "c1,c2",
    to       : "c3",
    sum      : 100
  });

  // COLUMN MODE: r4 = r1+r2+r3 (per column)
  autoSum({
    question : "#question_Sum1",
    mode     : "col",
    from     : "r1,r2,r3",
    to       : "r4"
  });
</script>
]]></style>
</number>
Configuration Options
ParameterTypeRequiredDescription
question string * jQuery selector for the question container
from string * Comma-separated source columns (row mode) or rows (col mode)
to string * Target column (row mode) or row (col mode)
sum number opt Limit value for calculation. If omitted, shows raw sum.
op string opt "subtract" (default) or "add"
mode string opt "row" (default) or "col"
rows string opt Comma-separated row labels to restrict (row mode only)
cols string opt Comma-separated column labels to restrict (col mode only)
💡
Auto-Detection: The component auto-detects the prefix from class names (e.g., Q1_r1_c1 extracts prefix Q1). Target inputs become readonly with gray background. Negative values show red background (#fdd).

  1. None Checkbox
    When "None" checkbox is checked, all .text-input values in that column are cleared.

    Auto-detects column count by scanning for col1, col2, ... classes.

Open End (OE)

Dynamic text input show/hide

How It Works

The OE system automatically shows open-end text fields when specific options are selected. It uses a class naming convention and moves the OE input into the selected option's container.

  1. Name Your OE Field
    Give the OE input a class following the pattern: {questionClass}_{optionClass}_showOE_1
  2. Add OE_hide Class
    Add OE_hide class to hide the OE initially.
  3. Auto-Behavior
    When option is selected, OE appears next to it. When deselected, OE hides and clears.

Piping

Transfer answers between questions

How It Works

Piping transfers answers from a source question to a target element. It supports all input types: radio, checkbox, text, number, and textarea. For radio/checkbox, it pipes the label text by default, or the OE value if an open-end field is filled.

Pipe Format
Syntax 
// Format: targetSelector:sourceQuestion
// Multiple pipes separated by commas

data-pipe=".pipe1:X1"

// The script finds the target element and sets its text to:
// - For radio/checkbox: the label text of checked items
// - If OE exists and has value: the OE text instead
// - For text/number/textarea: the input value
// Multiple checked items are joined with ", "
💡
Priority: If a checked option has an associated OE field with text entered, the OE value takes priority over the label text. This is useful for "Other (please specify)" options where you want to pipe the custom text.
